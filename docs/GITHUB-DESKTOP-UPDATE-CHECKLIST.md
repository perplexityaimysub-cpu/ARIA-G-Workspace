# چک‌لیست به‌روزرسانی با برنامه‌ی GitHub Desktop (GITHUB DESKTOP UPDATE CHECKLIST)

این چک‌لیست مراحل دستی و گام‌به‌گام به‌روزرسانی مخزن گیت‌هاب را با استفاده از نرم‌افزار GitHub Desktop تبیین می‌کند.

---

### مراحل اجرایی دقیق به‌روزرسانی:

1. **باز کردن نرم‌افزار:** برنامه‌ی **GitHub Desktop** را در ویندوز باز کنید.
2. **اتصال محیط کاری محلی:** از منوی بالای برنامه مسیر `File > Add local repository` را انتخاب کنید.
3. **انتخاب مسیر:** پوشه‌ی زیر را به عنوان مخزن انتخاب کنید:
   `C:\Users\Admin\Documents\ARIA-G-Workspace`
4. **بررسی تغییرات:** تب **`Changes`** را در پنل سمت چپ باز کرده و لیست تغییرات را به دقت مرور کنید.
5. **انتخاب فایل‌های هدف:** فقط تیک فایل‌هایی را فعال نگه دارید که مایل به انتشار آن‌ها در این نسخه آپدیت هستید.
6. **ممنوعیت ریست یا دیسکارد:** هرگز از گزینه‌ی **Discard Changes** برای فایل‌های اصلی پروژه، مدارک یا کاتالوگ‌های بایگانی شده استفاده نکنید، مگر اینکه مطمئن باشید می‌خواهید تغییرات آن فایل خاص را کاملاً لغو کنید و به نسخه قبلی بازگردانید.
7. **نوشتن یادداشت کامیت:** یک عنوان کوتاه و خلاصه تغییرات مناسب و واضح در کادر پایین بنویسید (مثال: "تعریف ۴ محصول تستی جدید و استراتژی متفرقه").
8. **کامیت محلی:** روی دکمه‌ی آبی رنگ **`Commit to main`** کلیک کنید.
9. **دریافت اطلاعات سرور:** روی دکمه‌ی **`Fetch origin`** در منوی بالا کلیک کنید تا وضعیت سرور با محلی سینک شود.
10. **آپلود نهایی:** در صورتی که سرور فاقد تغییر جدیدی بود یا برنامه اعلام کرد هیچ تعارضی وجود ندارد، روی دکمه‌ی **`Push origin`** کلیک کنید.
11. **بروز تعارض:** در صورتی که برنامه هشدار تعارض (Conflict) داد، بدون دستکاری یا حذف فایل‌ها، کار را متوقف کرده و درخواست بررسی یا ممیزی نمایید.
