# راهنمای همگام‌سازی دستی گیت‌هاب (MANUAL GITHUB SYNC)

این فایل راهنمای رسمی برای مدیریت و همگام‌سازی دستی مخزن گیت‌هاب محیط کاری فعال زنگ اول (ARIA-G) است.

---

## ۱. مشخصات عمومی مخزن
* **وضعیت محیط کاری:** این پوشه (`C:\Users\Admin\Documents\ARIA-G-Workspace`) پوشه اصلی و فعال پروژه (Primary Active Workspace) است.
* **وضعیت همگام‌سازی خودکار:** **هیچ‌گونه همگام‌سازی خودکار یا پردازش پس‌زمینه‌ای وجود ندارد.** ارتباط با گیت‌هاب کاملاً دستی و تحت کنترل مستقیم اپراتور است تا از تداخل ناخواسته داده‌ها جلوگیری شود.

---

## ۲. روش همگام‌سازی پیشنهادی (GitHub Desktop)
استفاده از رابط کاربری گرافیکی **GitHub Desktop** برای آپدیت‌ها و مرور تغییرات توصیه می‌شود:
1. نرم‌افزار **GitHub Desktop** را باز کنید.
2. از منوی بالا گزینه‌ی `File > Add Local Repository...` را انتخاب کرده و آدرس این محیط کاری را انتخاب کنید.
3. در تب `Changes` تمامی فایل‌های تغییر یافته را بررسی کنید.
4. فایل‌های موقت، پوشه `scratch` و کش‌ها به طور خودکار طبق فیلترهای `.gitignore` نادیده گرفته می‌شوند.
5. پس از تایید فایل‌ها، یک خلاصه تغییرات بنویسید و دکمه‌ی `Commit to main` را کلیک کنید.
6. دکمه‌ی `Fetch origin` را بزنید تا تغییرات احتمالی گیت‌هاب بررسی شود.
7. در صورت عدم وجود تداخل، دکمه‌ی `Push origin` را کلیک کنید تا تغییرات به گیت‌هاب منتقل شوند.

---

## ۳. دستورات دستی Git در خط فرمان (PowerShell)
در صورتی که ترجیح می‌دهید از خط فرمان استفاده کنید:

* **مشاهده وضعیت تغییرات محلی:**
  ```powershell
  git status
  ```
* **بررسی آخرین کدهای ثبت شده روی سرور:**
  ```powershell
  git fetch origin
  ```
* **ثبت تغییرات جدید و آپلود دستی:**
  ```powershell
  git add .
  git commit -m "توضیح کوتاه تغییرات اعمال شده"
  git push origin main
  ```

## ⚠️ قوانین ایمنی بسیار مهم (Strict Safety Rules)
* **عدم حذف فایل‌های پروژه:** هرگز پوشه‌های `بایگانی/`، `اسناد/`، `خروجی/` یا پکیج‌های نصبی را برای خلوت کردن یا «تمیزکاری» دستی مخزن حذف نکنید. این فایل‌ها جزء جدایی‌ناپذیر کاتالوگ و مستندات سیستم هستند.
* **بروز تعارض (Conflict):** در صورتی که پس از دستور `Fetch` متوجه شدید تغییرات محلی با سرور تداخل دارند، از اجرای دستورات بازنویسی تاریخچه نظیر `git push --force` یا `git reset --hard` به شدت خودداری کنید. فرآیند را متوقف کرده و آن را ممیزی نمایید.
* **فایل‌های نادیده گرفته شده:** پوشه `scratch/` که حاوی ابزارهای کمکی و آزمایشی موقت شما است، برای جلوگیری از آلودگی مخزن اصلی در فایل `.gitignore` قرار گرفته و آپلود نخواهد شد.
