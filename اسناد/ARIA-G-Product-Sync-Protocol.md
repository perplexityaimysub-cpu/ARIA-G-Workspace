# پروتکل ممیزی و همگام‌سازی بایگانی دیتابیس کالاها (ARIA-G Database Sync Protocol)

این سند تعیین‌کننده نحوه تعامل کارفرما با سیستم خودمختار ARIA-G برای رفع نواقص احتمالی کالاهای کاتالوگ، افزودن تنوع‌های جدید و همگام‌سازی خودکار دیتابیس بایگانی با خروجی‌های شاپفا است.

---

## ۱. فلسفه طراحی حلقه بسته بازخورد (Closed-Loop Feedback)

به دلیل تنوع گسترده بازار نوشت‌افزار در ایران، ممکن است بعضی محصولات در زمان تعریف توسط سیستم ایجنت به دلایلی دارای نواقص اطلاعاتی باشند (مانند عدم تعریف تمام رنگ‌ها یا کل وزن‌ها). 
به منظور برطرف کردن دائمی این معضل، ابزار خودکارساز ممیزی و هماهنگ‌کننده تعاملی زیر طراحی شده است:

```
[بازخورد کارفرما: افزودن متغیر جدید] 
      │
      ▼
[اجرای ابزار همگام‌ساز: sync_product_archive.py]
      │
      ├─► به‌روزرسانی بانک اطلاعات فکت‌ها (بایگانی JSON)
      │
      └─► بازتولید آنی و خودکار خروجی‌های پورتال (داشبوردهای HTML)
```

با این مکانیزم، هر تغییری که روی متغیرها اعمال شود، بلافاصله در داشبورد HTML و آرشیو JSON ثبت شده و به پایگاه دانش سیستم فیدبک داده می‌شود تا در مراحل تحقیقات بعدی از همان داده‌های به‌روزشده استفاده کند.

---

## ۲. راهنمای استفاده از ابزار همگام‌ساز (Sync CLI Tool)

ابزار همگام‌ساز تعاملی در آدرس `scratch\sync_product_archive.py` مستقر شده است و به شما یا ایجنت‌ها اجازه می‌دهد با وارد کردن یک دستور ساده در ترمینال، کالاها را گسترش دهید.

### سناریوهای متداول همگام‌سازی:

#### الف) افزودن یک رنگ جدید به روان‌نویس کاندید:
اگر روان‌نویس کاندید ۰.۵ دارای رنگ جدیدی در بازار شد (مثلاً رنگ صورتی پاستلی با کد اختصاصی انبار)، کافی است دستور زیر را در ترمینال پوشه روت اجرا کنید:
```bash
python scratch\sync_product_archive.py --slug c-class-candid-gel-pen-05 --action add_variant --var_type "رنگ جوهر" --var_val "صورتی پاستلی" --var_sku "110-110-0500-105"
```

#### ب) افزودن یک وزن جدید به چسب ماتیکی سی‌کلاس:
اگر وزن جدیدی (مثلاً ۴۰ گرم) برای چسب ماتیکی تولید و عرضه شد:
```bash
python scratch\sync_product_archive.py --slug c-class-glue-stick-36g --action add_variant --var_type "وزن کالا" --var_val "۴۰ گرم" --var_sku "110-310-0040-040"
```

#### ج) به‌روزرسانی یکی از مشخصات فنی (مشخصات محصول):
اگر بخواهید یکی از ردیف‌های جدول مشخصات فنی (مثلاً جنس بدنه روان‌نویس مورنو) را در کل دیتابیس اصلاح کنید:
```bash
python scratch\sync_product_archive.py --slug c-class-moreno-soft-touch-gel-pen-07 --action update_attribute --attr_name "جنس بدنه" --attr_val "پلاستیک فشرده ABS با پوشش مخملی"
```

## ۳. نحوه انتقال تغییرات و ممیزی توسط کارفرما

هرگاه شما به عنوان ناظر ارشد، تغییری را روی کالاها اعمال کرده یا نواقصی را برطرف کردید، در درخواست بعدی خود صرفاً کافی است عبارت زیر را به هوش مصنوعی (Antigravity) اعلام کنید:
> "کالای با اسلاگ `[SLUG]` را با افزودن تنوع `[VALUE]` با کد انبار `[SKU]` به‌روزرسانی کردم. بایگانی را بازبینی و همگام‌سازی کن."

سیستم بلافاصله فایل‌های JSON بایگانی اولیه را با خروجی‌های شاپفا منطبق خواهد کرد تا پایداری و صحت کاتالوگ حفظ گردد.
